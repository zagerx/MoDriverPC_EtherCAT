# 当前任务

## 已完成

### 1. 扩展 Activity 框架 ✅

- `SdoParameterActivity`：通过 SDO 写入参数并可选回读校验，失败策略 `kEnterError`。
- `StateInspectionActivity`：读取并记录从站实际 EtherCAT 状态与 AL status code，失败策略 `kKeepControllerState`。
- `CheckSlaveStates()` 已扩展为覆盖 `kMaintenance`（原 PreOp）和 `kOperational` 两种稳定状态的掉线检测。

### 2. 命令交互简化 ✅

- 从复杂的事件/管道架构简化为基于 stdin 的交互式命令行。

### 3. switch-case 主循环 ✅

- 实现 `EcatApplication::Run()` switch-case 主循环。
- 实现各状态 `HandleXxxState()` 函数。
- 使用 `poll()` 实现 `ReadCommand()` 100ms 超时读取 stdin。
- `main.cpp` 注册 `SIGINT` / `SIGTERM` 信号处理。

### 4. 精简 ControllerState 为 6 状态 ✅

- `kUninitialized` / `kAdapterReady` / `kScanned` / `kMaintenance` / `kOperational` / `kError`
- 删除 `kPdoConfigured` / `kSafeOp` / `kDcConfigured`，作为 `StartOperation()` 内部子步骤。
- `kInitDone` → `kAdapterReady`，`kPreOp` → `kMaintenance`。
- 更新 `TransitionTo()`、`kAllowedTransitions`、公共接口、Activity `CanStart()`。

### 5. CommandReader 抽象 ✅

- 新增 `CommandReader` 接口，返回 `ReadResult`（kOk / kTimeout / kEof / kError）。
- 新增 `StdinCommandReader` 实现 stdin + poll 读取。
- `EcatApplication` 通过构造函数注入 `CommandReader`，不再依赖 stdin 细节。
- `main.cpp` 注入 `StdinCommandReader`。

## 当前目标

等待硬件测试验证：

1. 正常 PREOP 维护流程：`scan -> config -> diagnose -> inspect -> stop -> exit`
2. 状态错误命令过滤：在错误状态下输入非法命令应报错
3. PREOP 掉线检测：在 `kMaintenance` 状态下拔掉从站网线，观察是否进入 `kError`
4. 退出方式：`exit` / `Ctrl+D` / `Ctrl+C` 都能正常退出
5. 重新扫描：`stop` 后能再次 `scan -> config`

## 验收标准

1. `ecat_scan` 编译通过。
2. 从站能进入 PREOP（`kMaintenance`）并执行 SDO 诊断。
3. 命令按新状态过滤，非法命令给出明确错误提示。
4. `kMaintenance` 状态下周期性调用 `CheckSlaveStates()`。
5. `Ctrl+C` 和 `exit` 都能正常退出。
6. 不进入 SAFEOP/OP，不启动独立 PDO 周期线程（除非显式测试 `start`）。
